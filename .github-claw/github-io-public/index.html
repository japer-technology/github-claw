What is github-claw?

github-claw is a self-contained AI agent that you drop into any GitHub repository as a single .github-claw/ folder. Every issue you open becomes a chat thread with an AI agent. The agent reads the issue, thinks, and replies as a comment — with 👀 shown while it works.

Conversation history is committed back to git after every turn, giving the agent long-term, versioned memory with no external database required. Powered by pi coding agent.

How it works

  1. 1
    Open an issue GitHub Actions triggers the agent workflow automatically.
  2. 2
    Agent reacts with 👀 Visual feedback that it's picked up the issue before any heavy computation starts.
  3. 3
    Agent thinks and responds The AI reads your issue (with full prior session context), acts on it, then posts a reply as a comment.
  4. 4
    State is committed to git The session and any file changes are pushed back to the repo — versioned, auditable, recoverable.
  5. 5
    Comment to continue Reply on the same issue to resume the conversation. The agent loads that session and picks up where it left off.

Repo as storage

All state lives directly in the repository — no database, no session cookies, just git:

.github-claw/state/
  issues/
    1.json          # maps issue #1 → its session file
  sessions/
    2026-02-04T..._abc123.jsonl    # full conversation for issue #1

Each issue number is a stable conversation key. When you comment on an issue weeks later, the agent loads the linked session and continues. Every word is in version control.

Setup — add to any repo in 5 minutes

  1. 1
    Copy the .github-claw folder into your repo root.
  2. 2
    Run the install script 
    bun .github-claw/install/github-claw-INSTALLER.ts
  3. 3
    Install dependencies 
    cd .github-claw && bun install
  4. 4
    Add your LLM API key Go to Settings → Secrets and variables → Actions and create a secret for your chosen provider.
  5. 5
    Commit, push, and open an issue The agent starts automatically.

Security

The workflow only responds to repository owners, members, and collaborators. Random users cannot trigger the agent on public repos.

The agent uses a fail-closed guard: every workflow run checks for the sentinel file github-claw-ENABLED.md. If it's missing, the workflow exits immediately. Delete or rename that file to disable the agent without removing any code.
